根因分析 / Root Cause
SnakeYAML 1.33 及之前版本默认允许通过全局 tag（如 !!java/javax.script.ScriptEngineManager）在 YAML 中指定任意 Java 类进行实例化。Composer 在解析 tag 时未进行安全检查，BaseConstructor 通过 yamlMultiConstructors 中的 !!java/ 前缀匹配规则无条件信任 tag 内容，直接调用 Class.forName() 加载并通过反射实例化对象。这使得攻击者可以构造恶意 YAML 实例化危险类（如 ScriptEngineManager、C3P0 连接池等），触发类加载、静态代码块执行或 setter 注入，最终实现 RCE。
漏洞代码走查 / Code Walkthrough 4 ENTRIES
Composer.composeScalarNode: nodeTag = new Tag(tag);
Composer 从 YAML 解析出 tag 后直接封装为 Tag 对象，未进行任何安全校验，允许任意全局 tag 通过
BaseConstructor.getConstructor: for (String prefix : yamlMultiConstructors.keySet()) { if (tag.startsWith(prefix)) { return yamlMultiConstructors.get(prefix); } }
通过前缀匹配查找 Construct。Constructor 类在初始化时注册了 !!java/ 前缀对应 ConstructYamlObject，导致所有 !!java/ 开头的 tag 都会被处理
Constructor$ConstructYamlObject.construct: Class<?> cl = getClassForName(name);
从 tag 中提取类名，通过 Class.forName() 加载类，无任何类白名单限制
Constructor$ConstructYamlObject.construct: return newInstance(...);
通过反射调用构造函数或 setter 方法实例化对象，触发恶意类的代码执行
修复措施
补丁引入了 TagInspector 机制对全局 tag 进行白名单校验。LoaderOptions 中新增 tagInspector 配置（默认为 UnTrustedTagInspector，拒绝所有全局 tag）。Composer 在解析 tag 时通过 isGlobalTagAllowed() 检查，未通过则抛出 ComposerException。
修复完整性
修复较为完整。默认 deny-all 策略有效阻止任意类实例化。但需注意：用户若自定义 TagInspector 并配置过于宽松的白名单仍可能引入风险；现有代码升级到 2.0 后可能因默认拒绝全局 tag 而出现兼容性问题。

前置条件
  • 应用依赖 SnakeYAML <= 1.33
  • 应用使用默认 Constructor（非 SafeConstructor）解析外部输入的 YAML
  • 攻击者可控制 YAML 输入内容（HTTP body、消息队列、配置文件等）
  • Classpath 中存在可利用的 gadget 类（如 javax.script.ScriptEngineManager、C3P0、Spring 等）
后果
  • 远程代码执行（RCE）：通过实例化恶意类执行任意代码
  • 数据窃取：读取服务器文件、环境变量、数据库凭证等
  • 服务器控制：反弹 shell、植入后门
  • 拒绝服务（DoS）：实例化消耗大量资源的类
真实利用场景
  • Spring Boot 应用使用 @RequestBody 接受 YAML 格式的 HTTP 请求，攻击者提交包含 !!javax.script.ScriptEngineManager 的 payload 触发 RCE
  • Kafka/RabbitMQ 消费者使用 SnakeYAML 解析消息内容，攻击者向队列投递恶意 YAML 消息实现无交互 RCE
  • Kubernetes Operator 或配置中心使用 SnakeYAML 解析用户提交的配置文件，攻击者通过上传恶意 YAML 获取集群权限
违反的原则
  • 最小权限原则：默认允许实例化任意类，而非白名单模式
  • 输入验证：未对外部输入的 tag 进行安全校验
  • 安全默认值：默认 Constructor 应采用安全模式
  • 深度防御：反序列化层面未引入类白名单、对象过滤等多层防护
  • fail-safe 设计：遇到未知 tag 应拒绝而非尝试加载
修复建议
  • 立即升级到 SnakeYAML 2.0 或更高版本
  • 优先使用 SafeConstructor：new Yaml(new SafeConstructor())，彻底禁止全局 tag
  • 若必须支持自定义类，显式配置 TagInspector 白名单
  • 避免直接解析不可信来源的 YAML（外部 HTTP 请求、消息队列等）
  • 采用 JSON 等更安全的数据格式替代 YAML
类似模式
  • Jackson polymorphic deserialization（CVE-2017-7525）：通过 @class 字段指定任意类反序列化
  • Fastjson AutoType RCE（CVE-2017-18349）：通过 @type 字段实例化任意类
  • XStream 反序列化 RCE（CVE-2013-7285）：XML 中通过类型标签实例化任意类
  • Java 原生反序列化 RCE（Apache Commons Collections gadget chain）
